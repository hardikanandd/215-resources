Implementation Details

Installing OpenSSL to ensure availability of cryptographic libraries on systems running Solaris 2.x.
Applies to the practice:
Configure computers for secure remote administration
Offer only essential network services and operating system services on the server host machine
Keep operating systems and applications software up to date
Configure computers for file backups

Applicable technologies:
Sun Solaris 2.x; UNIX operating systems and derivatives
 
Cryptography is an essential element in securing communications and applications in public networks. Although it is a cornerstone in a society increasingly reliant on e-commerce, cryptography is a difficult discipline to master and is best left to professional cryptography developers for both algorithm development and implementation. Successful application developers should rely on the tools created by the cryptography professionals experts when writing software that uses cryptography. 

OpenSSL was developed to provide developers and administrators with a complete and extensive library for all important and frequently used cryptographic algorithms used in today's programs.  OpenSSL also provides programs that can be used to implement certification authorities, create private/public key pairs for S/MIME users or SSL servers, etc. These latter topics are beyond the scope of this implementation. 

This implementation describes how to install OpenSSL on the Sun Solaris operating system, version 2.x and is included for use with the implementation Installing, configuring, and operating the secure shell (SSH) on systems running Solaris 2.x. You can use this approach with other UNIX operating systems and hosts.
Effort Estimates The time needed to retrieve the source code from the distribution site varies depending upon your connection speed. The source code is less than 1 MB (version 0.9.6). Building and installing OpenSSL depends upon machine performance but on average takes less than thirty minutes. The majority of this time is needed to compile the source code. 
System Prerequisites You can build and install OpenSSL on many UNIX systems. 

Software Prerequisites
To build OpenSSL on your system, you need Internet access to retrieve the following software tools: 
  • An MD5 cryptographic checksum program 
  • GZIP to uncompress the downloaded file
  • A C compiler; either the Sun C Computer or the free GNU C Compiler 
Downloading and Verifying The latest release of OpenSSL is available from: 

ftp://ftp.openssl.org/source/. 

Download the distribution package (at the time of this writing openssl-0.9.6.tar.gz) into a directory (e.g., /tmp). 

To verify the authenticity with PGP, you also need to download the detached PGP signature file into the same directory from: 

ftp://ftp.openssl.org/source/openssl-0.9.6.tar.gz.asc
Verify the authenticity of the OpenSSL distribution with PGP.

To do so, you need the PGP public key from "Richard Levitte <levitte@lp.se>" within your own PGP public key ring. You can get a copy of his key from the public PGP key servers. 

A match with his digital signature ensures that the software was downloaded successfully without modification. To verify the digital signature, the following commands are issued in the directory containing the downloaded files: 

$ pgp openssl-0.9.6.tar.gz.asc

If the signature doesn't match, verify that 

  • your downloaded filenames match those listed above
  • your have included the needed PGP key in your public key ring
Verify the authenticity of the OpenSSL distribution with MD5.

If you haven't checked the authenticity with PGP you might verify the distribution with checksums using MD5. 
 
File to Download MD5 Checksum
ftp://ftp.openssl.org/source/openssl-0.9.6.tar.gz 4b407ab005b3846ec542eb8305823bca

If the computed cryptographic checksum of the download files does not correspond to the information above, verify that 

  • your downloaded filenames match those listed above
  • your checksum program computes MD5 message digests 
Building OpenSSL is distributed as a compressed source code package.
Unpack the OpenSSL distribution.
  The compressed file can be uncompressed using the GNU gunzip utility and unpacked using the system tar command: 

$ gunzip openssl-0.9.6.tar.gz
$ tar xvf openssl-0.9.6.tar

This command creates a subdirectory named openssl-0.9.6. All operations are to be performed within this subdirectory.
Build the OpenSSL programs and libraries.
  A configuration script config, provided with the distribution, generates a Makefile that contains all information to build and installs OpenSSL. 

You don't need to specify any option when executing the config script: 

$ sh ./config

After the configuration is finished, you can build the binaries and libraries by specifying the make command:

$ make
Testing
Test the OpenSSL programs and libraries before installation.
  OpenSSL comes with an extensive test suite for each algorithm implemented within the libraries. This is especially important because all other applications depend on these algorithms to ensure confidentiality and authenticity. To run all tests before the installation, execute the make command with the argument test: 

$ make test

All tests should execute with no errors. If an error occurs, the problem is likely with the configuration. Execute the installation procedure again from the beginning. If an error still occurs, verify the configuration for your system manually by checking the Makefile.
Installation
Install all programs and libraries into the correct system directories.
  It is difficult to perform the installation manually. Therefore, we recommend that you use the installation part of the Makefile after having reviewed it's content. To install OpenSSL, start the make command with the argument install: 

# make install

Once this command terminates, all files should reside in the correct system directory. You can then install other programs like OpenSSH that depend on the OpenSSL library.

Creating a Spam Filter Rule in Pine
 

CSD has implemented a new way of marking spam messages. Potential spam or junk mail messages are flagged in the subject line of each message with a "SpamScore". This allows USC e-mail users to filter spam and junk mail messages away from their inbox. For more information about CSD's effort against spam go to www.csd.sc.edu/emailservices/spam.shtml

Creating a Spam Filter

Pine's built-in filter function is the natural solution for Pine users since it is readily available, easy to use and offers more customization options than most clients.

1. To set up a filter, type the first letter of each command below:

(M)ain
(S)etup
(R)ules
(F)ilter
(A)dd

2. Then hit enter and name your filter. Something like "possible spam" is a good descriptive name. Enter again to save the name.

3. Now scroll down to just below FILTERED MESSAGE CONDITIONS BEGIN HERE and place your cursor on Subject pattern and hit Enter.

4. In the space provided, enter the following, exactly as it reads here (and then enter):

{SpamScore: sss

5. Scroll down to Message is New? and type a * on Yes.

6. Scroll down to ACTIONS BEGIN HERE. Under Filter Action put the cursor on Folder List = and hit enter. Enter the name of the folder where you want to put possible spam, such as "spam" or "potential-spam" to remind yourself that it could contain false positives.

7. Then, to save:

(E)xit Setup
(Y)es -- to save
(Y)es -- to create new folder
(E)xit Setup
(Y)es -- to save

8. Your new spam filter rule is saved.

9. Remember to check this spam or potential-spam folder regularly to catch any legitimate mail incorrectly marked as spam. Also be sure to delete the rest of the spam which collects there, so your quota is not exceeded.

Using Procmail Instead of Pine

If you already have a procmail filter and would rather use that for filtering potential spam, simply add some recipes to your procmailrc— see the examples below. However, if you prefer to use Pine's filter, you can do so without changing anything in procmail.

.procmailrc example

# Comments begin with #
# Messages with a score of 5 or
# higher are almost certainly spam.
# Send to almost-certainly-spam folder.

:0:
* ^{SpamScore: sssss
almost-certainly-spam

end .procmailrc example

As above with Pine filtering, you can use either or both of these filtering rules. For both, be sure to keep them in the order shown, or the X-Spam-Level rule will not be applied.
